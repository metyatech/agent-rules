## 概要

このリポジトリは、教材サイトなどの「利用側リポジトリ」から使われる再利用可能な Node モジュール（React コンポーネント / ユーティリティ）を想定する。

## 目的

- 教材サイト側の複雑さを増やさず、共通機能を共有する。

## 設計方針

- 「理想状態（分かりやすさ・安全性・保守性）」を優先する。
  - 破壊的変更が必要なら許容し、利用側リポジトリも同じ変更セットで追従させる。
- コース固有の事情（文言・URL・組織名・提出物の仕様など）をモジュールへ混ぜない。

## 設定・エラー

- オプションを受け取るプラグインは、起動時に設定を検証し、エラーは「直し方」が分かる形で出す（サイレントなフォールバックは避ける）。
- 認証や外部サービス連携を扱う場合は、ログ/例外に秘密情報を含めない。

## 公開（GitHub / npm）

- 公開予定のパッケージは最低限 `LICENSE` を含める。
- `files` を設定して配布物を限定し、不要なファイル（例: `node_modules/`, テスト生成物, ローカル設定）を含めない。
- スコープ付きパッケージを npm 公開する場合は `publishConfig.access: "public"` を設定する。
- 利用側が「クリーン環境の `npm install`」だけで使える状態を担保する（必要なら `prepare` 等でビルドする）。
- 公開内容が変わる場合は `version` を更新し、変更点が追えるようにする。

## 検証

- 配布物の想定がある場合は `npm pack --dry-run` で内容を確認する。
- テストがある場合は `npm test` を実行する。
